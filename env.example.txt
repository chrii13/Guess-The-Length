# ==========================================
# Note Importanti
# ==========================================
#
# 1. Chiavi Pubbliche (NEXT_PUBLIC_*):
#    - Sono esposte nel bundle client di Next.js
#    - L'anon key di Supabase è progettata per essere pubblica
#    - Sono protette da Row Level Security (RLS) di Supabase
#
# 2. Chiavi Private (senza NEXT_PUBLIC_):
#    - NON sono esposte al client
#    - Disponibili solo in API routes server-side
#    - Service Role Key ha accesso completo - NON condividerla!
#
# 3. Rate Limiting:
#    - Valori più bassi = più restrittivo
#    - Valori più alti = più permissivo
#    - Considera l'uso previsto dell'API
#
# 4. Body Size:
#    - Limiti più bassi prevengono attacchi DoS
#    - Limiti più alti permettono payload più grandi
#    - 1KB è sufficiente per email check
#    - 1MB è un buon default generale
